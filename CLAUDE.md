## Commands

**Rebuild Tailwind CSS** (run after any HTML class changes):
```bash
npx tailwindcss -i assets/css/input.css -o assets/css/tailwind.css
```

Watch mode for active development:
```bash
npx tailwindcss -i assets/css/input.css -o assets/css/tailwind.css --watch
```

**Utility scripts** (run from project root):
```bash
node rewrite-layout.js   # Regenerates the 7 service sections in index.html from its embedded JS data array
node rewrite-promos.js   # Regenerates the promotions section in index.html from its embedded JS data array
node switch-icons.js     # Replaces Remix Icon tags with inline Lucide SVGs (one-time migration helper)
```

There are no tests and no linter configured.

## Architecture

Static single-page marketing website (Spanish) for a beauty salon/spa. No backend, no bundler, no framework â€” HTML, vanilla JS, and Tailwind CSS.

### Content pipeline

**Service sections and the promotions section are code-generated.** `rewrite-layout.js` contains a hardcoded `services` array (7 entries: BarberĂ­a, UĂ±as, Faciales, DepilaciĂłn, ArmonizaciĂłn Facial, Lashista & Cejas, Alaciados & Keratina) and rebuilds those sections in `index.html`. `rewrite-promos.js` contains a `promos` array (22 entries) and rebuilds the promotions section. To change prices, descriptions, or promos, edit the data arrays in the respective script and re-run it â€” do not edit those sections directly in `index.html`.

Services with a `promos` field (BarberĂ­a, Faciales, ArmonizaciĂłn Facial) render an "Ofertas del mes" strip inside their section showing additional deals not listed in their main price table. The strip is generated by `buildPromoStrip()` in `rewrite-layout.js`. Each promo card in `rewrite-promos.js` automatically computes a `-%` discount badge and "Ahorras $X" label from `oldPrice`/`price` when both are plain numbers.

The script locates each section by its `<!-- ===== TITLE ===== -->` comment marker. **Never remove or edit those comment markers manually** â€” if the regex fails to match, the script falls back to a weaker `<section id="...">` match and logs `"Could not replace <id>"` without erroring.

Six of the seven services use a flat `prices: []` array. The `depilacion` service instead uses `blocks: [{subtitle, prices[]}]` to render separate Hombre/Mujer subsections â€” match that shape if adding a similarly grouped service.

Both `rewrite-layout.js` and `switch-icons.js` read SVG files from `node_modules/lucide-static/icons/` at run time.

Other sections (navbar, hero, promotions grid, contact, footer) live only in `index.html` and must be edited there directly.

### CSS pipeline

`assets/css/input.css` (2-line entry point using **Tailwind v4** `@import "tailwindcss"` + `@config` syntax) â†’ Tailwind CLI â†’ `assets/css/tailwind.css` (committed, auto-generated). Custom components and overrides go in `assets/css/styles.css`. Font-face declarations are in `assets/css/fonts.css`.

`tailwind.config.js` defines the design tokens:
- **Colors:** `gold` (#C9A84C + light/dark variants), `dark` (#0A0A0A / card / surface / border), `warm` (gray / light / white)
- **Fonts:** `heading` = Cormorant Garamond (serif), `body` = DM Sans (sans-serif)

Always rebuild `tailwind.css` after adding new Tailwind utility classes to HTML.

### JS layer (`assets/js/main.js`)

Plain script, no modules, runs on `DOMContentLoaded`. Six independent features:

| Feature | Key selectors |
|---|---|
| Mobile drawer | `#menu-toggle`, `#menu-close`, `#mobile-drawer`, `#mobile-overlay` |
| Navbar scroll | `.navbar` â†’ adds `.scrolled` class at >60px scroll |
| Scroll reveal | `.reveal`, `.reveal-left`, `.reveal-right` â€” IntersectionObserver, one-time animation |
| FAQ accordion | `.faq-item` / `.faq-question` / `.faq-answer` â€” one-open-at-a-time |
| Promo filter | `.promo-filter-btn` + `.promo-card[data-category]` â€” show/hide by category |
| Hero canvas | `#hero-canvas` â€” `requestAnimationFrame` loop drawing rising gold particles and bokeh orbs |

Global `window.openWhatsApp(service)` pre-fills a WhatsApp message to +52 777 159 4874. Every price card calls this function.

### Icons

All icons are inline SVGs embedded directly in HTML with classes `inline-icon` or `section-icon`. `switch-icons.js` is a one-time migration tool that reads from `node_modules/lucide-static/icons/`. Remix Icon references may still exist in sections not yet migrated.
